# contour_asset.rs

## Purpose
Provides the globe renderer with local contour overlays around the selected focus point. It now loads bounded, cached SRTM contour tiles from the shared SQLite cache generated by `srtm_focus_cache.rs` and falls back to bounded GEBCO GeoPackage queries only when SRTM is unavailable.

## Components

### `ContourPath`
- **Does**: Holds one contour polyline and its elevation value
- **Interacts with**: `globe_scene.rs`

### `load_srtm_for_focus`
- **Does**: Loads locally cached SRTM contour vectors for one focus/zoom bucket and returns simplified contour polylines
- **Interacts with**: `srtm_focus_cache.rs`, `globe_scene.rs`
- **Rationale**: Keeps contour rendering on validated GDAL-derived assets instead of the broken runtime TIFF decode path while reading from one indexed tile store instead of one file per bucket

### `load_srtm_region_for_view`
- **Does**: Loads the current neighborhood of SRTM contour buckets around the local viewport center, retains previously visited buckets for the current selected-event scene, and returns one merged contour set
- **Interacts with**: `srtm_focus_cache.rs`, `local_terrain_scene.rs`
- **Rationale**: Prevents already visited terrain chunks from disappearing as the analyst pans across the streamed local map

### `load_for_focus`
- **Does**: Resolves the correct GEBCO contour asset for the current zoom, caches a local query window, and returns fallback contour polylines near the selected focus point
- **Interacts with**: `terrain_assets.rs`, `globe_scene.rs`
- **Rationale**: Preserves a low-resolution global fallback when SRTM tiles are missing or the focus point is outside land coverage

### `load_global_coastlines`
- **Does**: Loads a coarse `0m` global coastline layer for low-zoom globe rendering and now triggers a one-time background build from raw GEBCO tiles when the derived cache is missing
- **Interacts with**: `terrain_assets.rs`, `globe_scene.rs`
- **Rationale**: Gives the zoomed-out globe recognizable land/ocean boundaries without depending on blurry raster relief alone

### `query_local_contours`
- **Does**: Reads one locally cached focus tile from the shared SQLite contour cache and applies feature thinning / point simplification
- **Interacts with**: `srtm_focus_cache.rs`, `rusqlite`

### GeoPackage parsing helpers
- **Does**: Query the GEBCO GeoPackage RTree index and parse GeoPackage/WKB line geometries into geographic point lists
- **Interacts with**: `rusqlite`, derived contour assets under `Derived/terrain`

### Clipping helpers
- **Does**: Clip fallback GEBCO contour polylines to the local focus bounds before rendering
- **Interacts with**: `load_for_focus`
- **Rationale**: Prevents long global contour features from wrapping around the globe and dominating performance

## Contracts

| Dependent | Expects | Breaking changes |
|-----------|---------|------------------|
| `globe_scene.rs` | `load_for_focus` returns reasonably bounded fallback contour sets for the selected area | Returning full-world contours or changing the point representation |
| `local_terrain_scene.rs` | `load_srtm_region_for_view` preserves already visited chunk tile keys for the current selected-event scene while adding newly streamed ones | Removing scene retention or changing the merged contour contract |
| Terrain pipeline | Derived GEBCO contour layers remain in the `contour` GeoPackage layer | Renaming layer/table layout without updating SQL |

## Notes
- SRTM-generated contours are intentionally local and zoom-dependent; they are not meant to cover the whole visible globe.
- The SRTM contour path is now asynchronous at generation time: the first frame for a new focus/zoom bucket may show no contours until the cache file is built.
- The local-scene SRTM loader retains previously visited bucket keys per selected-event scene, so panning expands the streamed map instead of replacing it every time the viewport center moves.
- Scene retention is now keyed to the actual contour-cache zoom bucket instead of raw fractional zoom, which avoids unnecessary resets while the analyst is still within the same underlying terrain LOD band.
- The shared SQLite cache means all visited tiles live in one indexable file, which is a much better stepping stone toward global terrain streaming than the old directory of tiny GeoPackages.
- The zoomed-out globe still prefers `Derived/terrain/gebco_2025_coastline_0m.gpkg`, but it can now bootstrap that file from raw GEBCO topography tiles on a fresh machine instead of staying blank until someone manually copies a derived cache.
- The global coastline LOD now keeps the longest simplified features first instead of thinning by feature index, which avoids pathological cases where whole continents disappear while tiny islands survive.
- The GEBCO loader stays on `500 m` contours until closer zoom, then switches to `200 m` for fallback detail.
- Query extent, feature cap, clipping, and point simplification all tighten as zoom increases to keep interaction responsive.
